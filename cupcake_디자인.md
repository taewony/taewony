# 컵케이크 앱 디자인

## 1. 앱 구조 및 기능

컵케이크 앱은 사용자가 컵케이크를 주문할 수 있는 간단한 앱입니다. 다음 화면으로 구성됩니다.

*   **시작 화면:** 사용자를 환영하고 주문을 시작하는 버튼이 있는 초기 화면입니다.
*   **맛 선택 화면:** 사용자가 컵케이크의 맛을 선택할 수 있습니다.
*   **픽업 날짜 선택 화면:** 사용자가 픽업 날짜를 선택할 수 있습니다.
*   **주문 요약 화면:** 주문 요약을 표시하고 사용자가 다른 주문을 할 수 있도록 합니다.

이 앱은 Jetpack Compose를 사용하여 빌드되었으며 Model-View-ViewModel(MVVM) 아키텍처를 따릅니다.

*   **View:** UI는 컴포저블 함수(`StartScreen`, `FlavorScreen` 등)로 빌드됩니다.
*   **ViewModel:** `OrderViewModel`은 주문 상태(수량, 맛, 날짜)를 보유하고 비즈니스 로직을 처리합니다.
*   **Model:** 데이터는 간단하며 `OrderViewModel`에서 직접 관리됩니다.

## 2. Nav3란 무엇인가요?

Nav3는 Jetpack Compose를 위한 선언적, 상태 기반 내비게이션 라이브러리입니다. 프래그먼트 및 내비게이션 그래프에 의존하는 기존 내비게이션 시스템과 달리 Nav3는 간단한 목록을 사용하여 백 스택을 관리합니다.

Nav3의 핵심 구성 요소는 다음과 같습니다.

*   **`NavDisplay`:** 백 스택의 상태에 따라 현재 화면을 표시하는 컴포저블입니다.
*   **`NavEntry`:** 백 스택의 화면을 나타내는 컴포저블을 감싸는 래퍼입니다.
*   **백 스택:** 내비게이션 상태를 보유하는 간단한 `MutableList`입니다.

## 3. 컵케이크 앱에 Nav3가 적용된 방식

컵케이크 앱은 Nav3를 사용하여 여러 화면 간의 내비게이션을 관리합니다.

*   **`CupcakeApp.kt`:** 앱 UI의 기본 진입점입니다. `mutableStateListOf<Any>`인 백 스택을 만들고 관리합니다.
*   **`CupcakeScreen.kt`:** 이 봉인된 인터페이스는 앱의 여러 화면을 객체(`Start`, `Flavor`, `Pickup`, `Summary`)로 정의합니다.
*   **`NavDisplay`:** `CupcakeApp` 컴포저블은 `NavDisplay`를 사용하여 현재 화면을 표시합니다. `entryProvider` 람다는 각 `CupcakeScreen` 객체를 해당 화면 컴포저블을 포함하는 `NavEntry`에 매핑합니다.
*   **내비게이션:** 내비게이션은 백 스택에서 항목을 추가하거나 제거하기만 하면 처리됩니다. 예를 들어 `Flavor` 화면으로 이동하려면 `backStack += CupcakeScreen.Flavor`를 사용합니다. 뒤로 돌아가려면 `backStack.removeLastOrNull()`를 사용합니다.

이러한 내비게이션 접근 방식은 한 곳에서 모두 관리되고 간단한 상태 관리를 사용하므로 간단하고 유연하며 이해하기 쉽습니다.
